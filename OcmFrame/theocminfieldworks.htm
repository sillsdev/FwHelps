The Outline of Cultural Materials in FieldWorks

Introduction

FieldWorks incorporates the Outline of Cultural Materials (OCM) and FRAME as a system for categorizing fieldnotes and language data.

  • Data categorization is the process of systematically referencing data so that it can be found and organized according to a previously devised conceptual framework.
  • Consistently recorded cultural observations are the raw data needed for language and cultural learning, research, and translation. Categorizing raw data with topical codes like the OCM builds into your data a powerful analytical tool for cultural discovery.

Choosing an OCM category in Fieldworks

There are several ways to determine which OCM categories to use to “categorize” an entry in the Data Notebook:

If this is your situation:

Do this:

In a hurry or unfamiliar with the OCM?

Tag entry with one of the eleven FRAME categories.

Uncertain which OCM code to use?

Select “More” in the OCM Chooser dialog box, click on the Index hotlink, then enter an approximate word in the Index entry box and choose a code from among the options provided.

Know generally how you want to categorize an entry?

Browse through the OCM topics list to find the appropriate category (either major or subcategory)

Know the topic and correct OCM code

Use the Data Notebook’s type-ahead feature and begin entering either the OCM number or title directly in the field.

 

Using coded entries in FieldWorks Data Notebook

FieldWorks Data Notebook provides several ways to use data that has been categorized with OCM codes.

Analysis Tools

Filters and Sort Methods are two analytical tools in FieldWorks that allow you to

  • Compile subsets of entries based on specific information (e.g. entries with a particular OCM category).
  • Arrange entries based on the information in a specified field (e.g. Anthropology Categories/OCM).
Sort Methods

Using this analytical tool, you can arrange your data based on the OCM categories included in the entries. Once entries are displayed sorted in this way, you can use this feature to note the range of topics you have covered in your data, and determine how many entries occur for each category.

You can also create more complex sort methods. For example, you could first sort your data by Location or Source, and then by OCM category in order to see what types of information you have collected from each place or person.

Filters

Filters display only those entries that match filter criteria. There are two kinds of filters:

  • Simple filters — display entries that match one filter criteria
    • Examples of simple filters using OCM categories:
    • display only entries classified with a specific OCM category
    • display only entires classified with a specific OCM category and all its related subcategories
    • display all entries that are not yet tagged with any OCM category

    Note: Filters can be built using criteria other than OCM categories (e.g. Location, Source, Date of Event)

  • Complex filters — display entries that match two or more filter criteria
    • Examples of complex filters using OCM categories:
    • display entries that contain multiple groupings of OCM categories
    • display entries which contain one or more OCM categories and were obtained from a specified source.

 

Views

There are three different ways to look at your data. Certain views are especially useful after the application of a Filter or a Sort Method.

Data Entry

Read through filtered entries one at a time. This is the view normally used to enter data.

Browse  

A bird's eye view of your data with entries displayed in rows; each row showing selected fields. When used in combination with Filters and Sort Methods, this view is a quick way to see what is there in your field notes without a lot of extra clutter.

Document

Read selected information from a continuous set of field notes. This is the easiest way to read through relevant information from your field notes without having to switch screens for each entry. Simply scroll through the entries as if they were a single document. When used in combination with Filters, this view shows only a subset of your field notes; in essence giving a summary report of field notes which meet the filter criteria.

Note: Your are not restricted to these three preconfigured views. You can modify this views, and you can create any number of customized views based on these three types — hiding and showing, reordering, or modifying the look of any field as needed.
